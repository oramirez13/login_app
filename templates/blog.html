💉 SQL Injection
Inyección de código SQL

Ocurre cuando una aplicación incluye datos no confiables en una consulta SQL sin validación. El atacante puede leer, modificar o eliminar datos, bypassear autenticación o ejecutar comandos en el servidor.

Ejemplo de payload:
' OR '1'='1' --

Categoría: Injection - OWASP Top 10 #3
Cross-Site Scripting (XSS)
Inyección de scripts en el navegador

Permite inyectar scripts maliciosos en páginas vistas por otros usuarios. Puede usarse para robar cookies de sesión, redirigir usuarios o ejecutar acciones en nombre de la víctima sin su consentimiento.

Ejemplo de payload:
<script>alert('XSS')</script>

Categoría: Injection - OWASP Top 10 #3
Cross-Site Request Forgery (CSRF)
Falsificación de peticiones

Engaña al navegador de una víctima autenticada para enviar peticiones maliciosas a una aplicación web. El servidor las acepta como legítimas porque provienen del navegador del usuario autenticado.

Defensa principal:
CSRF tokens en formularios

Categoría: Broken Auth - OWASP Top 10 #2
Self-XSS
Ingeniería social en la consola

El atacante convence a la víctima de pegar código JavaScript malicioso en su propia consola del navegador. El navegador lo ejecuta con los permisos del usuario, pudiendo robar sesiones o datos sensibles.

Firefox y Chrome muestran una advertencia antes de permitir pegar código en la consola precisamente para defenderse de este ataque.

Ejemplo de payload:
document.body.innerHTML.match(/FLAG\{.*?\}/g)

Categoría: Social Engineering - OWASP Top 10 #6
